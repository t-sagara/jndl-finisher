# jndl-finisher

[国立国会図書館 NDL OCR Lite](https://github.com/ndl-lab/ndlocr-lite) が出力した OCR 解析結果の JSON をプレインテキストに変換するツールです。

## 背景

NDL OCR Lite は画像からテキストを抽出し、レイアウトボックスとテキストブロックの階層構造を持つ JSON を生成します。
単純にテキストを並べるだけでは、ページ端で折り返されたブロックが別の文として改行されてしまいます。
このツールはバウンディングボックス情報をもとに行末の折り返しを検出し、自然なプレインテキストを生成します。

## 機能

- テキストブロックのバウンディングボックスを解析し、レイアウトボックス端で折り返されたブロックを自動的に結合
- 行間の空き（ギャップ）を比較し、通常より広い箇所に空行を挿入（段落区切りの検出）
- ページをまたいで末尾が切れている場合も、次ページ先頭と結合
- テキストのないページ（空ページ）は段落区切りとして処理
- 縦書き・横書きの両方に対応

## セットアップ

```bash
git clone <this-repo>
cd jndl-finisher
uv sync
```

## 使い方

### 単一ファイルを変換

```bash
uv run python main.py path/to/page.json
```

### ディレクトリ内の全 JSON を変換（ページ順に結合）

```bash
uv run python main.py path/to/json-dir/ -o output.txt
```

### オプション

| オプション | デフォルト | 説明 |
|---|---|---|
| `-o`, `--output` | 標準出力 | 出力先ファイル |
| `--threshold` | `1.0` | 端判定の許容幅（文字数で指定）。値を大きくすると端と判定しやすくなる |

`--threshold` はピクセル数ではなく文字数で指定します。ブロックのバウンディングボックスサイズと文字数から1文字あたりのピクセル数を算出し、`threshold × px/文字` 以内の余白であれば端と判断して次のブロックと結合します。

## 入力 JSON フォーマット

NDL OCR Lite が出力する形式を想定しています。

```json
{
  "contents": [
    [
      {
        "boundingBox": [[x1,y1],[x1,y2],[x2,y1],[x2,y2]],
        "text": "テキスト内容",
        "isVertical": "true",
        "isTextline": "true",
        "confidence": 0.95
      }
    ]
  ],
  "imginfo": {
    "img_width": 397,
    "img_height": 591
  }
}
```

`contents` の各要素がレイアウトボックスで、その中に1つ以上のテキストブロックが含まれます。
